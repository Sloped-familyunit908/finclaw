# 我做了一个 7x24 小时自进化量化策略引擎（开源）

## 背景

做量化的人都知道一个痛苦的循环：构思策略、写回测代码、调参数、跑出好看的收益曲线、上线后发现不行、继续调参。

这个循环的根本问题在于：人工调参效率低、容易过拟合、而且搜索空间太大。一个多因子策略可能涉及 RSI、MACD、布林带、PE、ROE、毛利率等几十个维度的参数组合，靠人工穷举是不现实的。

我花了几个月时间做了一个系统来解决这个问题：用遗传算法让策略自己进化，7x24 小时不间断运行，无需人工干预。

项目叫 FinClaw，MIT 协议开源，代码在 GitHub：https://github.com/NeuZhou/finclaw

## 核心思路

把每个交易策略抽象为一个 41 维向量（我称之为"策略 DNA"），每个维度对应一个因子的权重或阈值。然后用遗传算法的标准流程来搜索最优策略：

1. **初始化种群** -- 随机生成 30 个策略配置
2. **评估适应度** -- 对每个策略在 500+ 只 A 股上做 walk-forward 回测（70% 训练集 / 30% 测试集）
3. **选择** -- 保留测试集上夏普比率最高的 5 个策略
4. **繁殖** -- 通过交叉（两个策略交换部分因子权重）和变异（随机扰动参数）产生下一代
5. **循环** -- 每代耗时几分钟，每天可以跑几百代

## 41 维因子空间

策略 DNA 编码了三类因子：

**技术因子（11个）：** RSI、MACD、布林带、KDJ、OBV、ATR、ADX、ROC、CCI、MFI、Aroon

**基本面因子（8个）：** PE、PB、ROE、营收同比增长、营收环比增长、利润增长、资产负债率、股息率

**质量因子（3个）：** 毛利率、现金流质量、PEG

其余维度编码持仓周期、调仓频率、仓位管理规则、行业配置约束等元参数。

## 防过拟合

过拟合是量化研究的头号敌人。FinClaw 用了三层防线：

**Walk-Forward 验证：** 每个策略只在样本外数据上计算适应度。在训练集上表现再好、测试集上不行的策略会被自然选择淘汰。

**蒙特卡洛模拟：** 通过打乱交易顺序跑 1000 次模拟。如果策略的表现对交易顺序高度敏感，说明可能存在过拟合。

**种群多样性控制：** 变异算子确保种群不会过早收敛到局部最优。多样性指标会惩罚过于相似的种群。

## 进化结果：第 1,559 代

经过 1,559 代持续进化，最优策略在样本外测试集上的表现：

| 指标 | 数值 |
|------|------|
| 年化收益率 | 309.6% |
| 夏普比率 | 3.42 |
| 最大回撤 | 13.8% |
| 胜率 | 62.9% |

回测数据来自 A 股市场约 5000 只股票，数据源使用 BaoStock 和 AKShare（均为免费数据源，无需 API Key）。

需要强调的是：这是回测结果，不是实盘结果。实盘中的滑点、流动性约束、冲击成本都会降低实际收益。A 股的涨跌停限制和 T+1 交割制度也意味着这些结果不能直接照搬到其他市场。

## MCP 服务器：AI 原生接口

FinClaw 内置了 MCP（Model Context Protocol）服务器，可以把量化分析能力暴露给 Claude、Cursor 等 AI 工具。支持 10 个工具：

- 实时行情查询
- 历史 K 线数据
- 技术指标计算
- 多因子选股
- 组合分析
- 策略回测
- 情绪分析
- 策略对比
- 交易所列表
- 资金费率查询

配置方法：

```json
{
  "mcpServers": {
    "finclaw": {
      "command": "finclaw",
      "args": ["mcp", "serve"]
    }
  }
}
```

## 技术栈

- Python 3.9+，核心引擎和进化系统
- Next.js 仪表盘，集成 TradingView 图表
- 数据源：BaoStock、AKShare（A 股）、Yahoo Finance（美股）、Binance/Bybit/OKX（加密货币）
- 基础功能零 API Key，安装即用

## 快速开始

```bash
pip install finclaw-ai
finclaw demo          # 体验所有功能
finclaw quote 600519  # 查询贵州茅台实时行情
```

自己运行进化引擎：

```bash
python scripts/run_evolution.py --generations 999999 --population 30
```
