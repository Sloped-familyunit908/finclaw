# 我用遗传算法让交易策略自己进化了127代——这是结果

> 大多数量化交易工具让你写策略。FinClaw让策略自己进化。

## 起因

做量化的人都知道一个痛点：你得先有一个策略思路，然后编码，然后回测，然后调参，然后发现不行，然后重来。

这个循环我跑了大半年，跑烦了。

我开始想一个问题：策略本身能不能像生物一样进化？不是我告诉它"均线金叉做多"，而是让它自己去发现什么规则能赚钱。

这就是 FinClaw 的核心思路——用遗传算法让交易策略自我进化。

## 遗传算法 + 量化交易 = ?

先简单说下原理。遗传算法模拟自然选择：

1. **初始种群**：随机生成一批交易策略（每个策略由一组参数和规则组成）
2. **适应度评估**：在历史数据上跑回测，用收益率、夏普比率、最大回撤等指标打分
3. **选择**：表现好的策略活下来
4. **交叉**：两个好策略"杂交"，产生后代
5. **变异**：随机扰动一些参数，引入新可能性
6. **重复**：新一代继续淘汰、杂交、变异

听起来简单，但工程上有几个硬骨头。

### 484维因子空间

FinClaw 不是简单的"调调均线周期"。每个策略的搜索空间覆盖了 484 个因子维度，包括：

- 技术指标类：各周期的 MA、RSI、MACD、布林带等
- 量价关系类：成交量变化率、量价背离信号
- 波动率类：ATR、历史波动率、隐含波动率代理
- 时间类：日内时段、星期效应、月份效应
- 跨品种类：相关性因子、价差因子

每个因子都是策略基因组上的一个"基因位"。遗传算法在这个 484 维的空间里搜索最优组合。

### Walk-Forward 验证：对抗过拟合的武器

说到这里，做量化的朋友肯定要问：这不就是变相的数据挖掘吗？过拟合怎么办？

问得好。这也是我最花心思的地方。

FinClaw 采用 Walk-Forward 验证机制：

```
时间轴: ──────────────────────────────────────→

第1轮: [===训练===][=验证=]
第2轮:    [===训练===][=验证=]
第3轮:       [===训练===][=验证=]
...
```

每一轮进化都在一个训练窗口上做，然后在紧接着的验证窗口上检验。策略的适应度得分取的是**验证集上的表现**，不是训练集。

进化出来的策略如果只是在训练数据上长得漂亮，在验证集上一塌糊涂，照样被淘汰。

## 127代进化：到底发生了什么

直接上数据。我跑了一轮完整进化，市场选的是加密货币（波动大，信号多，适合遗传算法发挥）。

进化过程中，种群的平均 fitness 变化大致是这样的：

```
第  1 代: fitness 0.12  ← 随机策略，基本在零线附近晃
第 10 代: fitness 0.31  ← 开始出现有效信号的组合
第 30 代: fitness 0.58  ← 快速提升期，好基因在扩散
第 60 代: fitness 0.71  ← 提升放缓，开始精细优化
第 90 代: fitness 0.78  ← 进入平台期
第127 代: fitness 0.82  ← 收敛，继续跑下去提升很小了
```

几个有意思的观察：

1. **前30代提升最快**。这很符合遗传算法的规律——好的"基因模块"被迅速发现和传播
2. **60代以后进入微调阶段**。大的结构已经稳定，变异主要在参数层面做精调
3. **127代基本收敛**。后面跑了50代也没明显提升，说明搜索空间已经被充分探索

最终跑出来的最优策略，我不会公布具体参数（公布了也没用，市场在变），但可以说它的结构出乎我意料——它组合了几个我从来不会主动写在一起的因子。

这就是遗传算法的价值：**它不受人类认知偏差的约束。**

## 跑起来

FinClaw 是 Python 写的，安装很简单：

```bash
git clone https://github.com/NeuZhou/finclaw.git
cd finclaw
pip install -e .

# 启动进化
finclaw evolve --market crypto

# 指定进化代数
finclaw evolve --market crypto --generations 200

# 使用自定义配置
finclaw evolve --config my_config.yaml
```

进化过程中会实时输出每一代的最优 fitness、平均 fitness 和种群多样性指标。

如果你有自己的数据源，也可以对接：

```python
from finclaw.data import DataProvider
from finclaw.evolve import GeneticEvolver

# 自定义数据
provider = DataProvider(source="your_data.csv")

# 配置进化参数
evolver = GeneticEvolver(
    population_size=100,
    mutation_rate=0.05,
    crossover_rate=0.7,
    factor_dimensions=484
)

# 开跑
best_strategy = evolver.run(
    data=provider,
    generations=150,
    validation_method="walk_forward"
)
```

## 诚实说两句

我不会告诉你"用了 FinClaw 就能赚钱"。量化交易没有圣杯。

遗传算法的优势是搜索效率高、不受人类偏见限制。但它也有明确的局限：

1. **过拟合风险始终存在**。Walk-Forward 能缓解，但不能完全消除。市场的非平稳性意味着过去有效的模式未来未必有效
2. **计算成本不低**。484维因子 × 100个体 × 127代，每一代都要跑完整的回测。没有 GPU 加速的话，一轮完整进化跑下来要几个小时
3. **结果不可解释**。进化出来的策略是个黑盒。你知道它能赚钱（至少在验证集上能），但你不一定能理解为什么

这也是我做 Walk-Forward 的根本原因——既然不能完全理解它，至少得确保它不是在骗我。
